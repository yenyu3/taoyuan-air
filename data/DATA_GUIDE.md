# 資料目錄結構說明

## 📁 目錄用途

### `raw/` - 原始資料

存放從外部來源取得的原始資料，不應該被修改。

#### `epa-stations/` - EPA 測站資料

- 來源：環境部空氣品質監測網
- 格式：JSON 檔案
- 內容：桃園市 5 個測站的空氣品質小時值資料
- 時間範圍：2026年1-3月

### `processed/` - 處理後資料

存放經過清理、轉換或分析後的資料。

### `exports/` - 匯出資料

存放從資料庫匯出的資料，用於備份或分享。

## 📋 資料管理原則

1. **原始資料不可修改** - `raw/` 目錄下的檔案應保持原始狀態
2. **版本控制** - 重要的處理後資料應該有版本標記
3. **文檔記錄** - 每個資料集都應該有對應的說明文檔
4. **定期清理** - 定期清理不需要的臨時檔案

## 🔄 資料流程

```
外部來源 → raw/ → 資料庫 → processed/ → exports/
```
